Detailed Description

Svg asset global management system: First add Svg assets to the system through the RegisterSvgAssets method. It will try to asynchronously load the assets under the specified FPrimaryAssetType and try to Cast them as UScalableVectorGraphicsWrapper. After all this is over, OnAssetLoaded Delegate will be triggered. After all this, use the GetTextureFromSvg method to get the Svg-converted UTexture2D of the specified Id, which is the name of the imported file by default. All converted UTexture2D will be strongly referenced by the system. When you call the GetTextureFromSvg method multiple times, if the requested size has been created, the system will not NewObject. This system defaults to cleaning up all strong references after PreLoadMap broadcast. If you do not want the UTexture2D derived from some Svg assets to be GCed, you need to place these Svg assets in another path and create another one in the asset manager of the project settings. PrimaryAssetType and specify the path and Class The best way to use it is to use the SVG Image Widget, which will create a UTexture2D directly in the Editor design. You can turn off the preview function and it will communicate with the system while the game is running in order to get the best performance.
